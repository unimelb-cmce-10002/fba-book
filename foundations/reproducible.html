3  Reproducible Workflows with R & Posit Cloud

If science is about systematically building and organizing knowledge in the form of testable explanations and predictions, then business science applies this mindset to the world of data-driven business decision-making. A business scientist doesn‚Äôt just crunch numbers‚Äîthey seek to generate insights that are reliable, testable, and useful to others.

This means we don‚Äôt just ask questions and analyze data. We build knowledge that others can follow, test, and build upon. It‚Äôs not enough to reach the right conclusion once, in a way that only you understand. True business science requires reproducible workflows‚Äîways of working with data that are clear, transparent, and repeatable.

This chapter introduces the tools we‚Äôll use to begin doing just that: R, a language designed for data analysis, and Posit Cloud, a cloud-based environment for running R code. Together, they‚Äôll help us learn how to think and work like business scientists from day one.

Why Not Excel?

Spreadsheets like Excel or Google Sheets are familiar‚Äîand for many tasks, they work just fine. But they don‚Äôt scale well when you‚Äôre doing real analysis.

Here‚Äôs why we don‚Äôt use spreadsheets for this course:

  • No clear record of what you did
    In Excel, it‚Äôs easy to lose track of which formulas you changed or which filters you applied. With code, every step is written down.

  • Easy to make silent errors
    One wrong cell reference can quietly mess up your whole analysis. In R, mistakes are more visible‚Äîand easier to fix.

  • Hard to repeat or update
    If you need to run the same analysis again with new data, you‚Äôll likely have to repeat every step by hand. With code, one click reruns the whole process.

  • Not designed for teamwork
    It‚Äôs difficult to collaborate on a spreadsheet without overwriting someone else‚Äôs work. With code-based tools, versioning and sharing are built in.

We‚Äôre not anti-Excel. But to become a business scientist, you need tools that are transparent, reproducible, and powerful. That‚Äôs why we start with R.

3.2 What is RStudio?

When you open your project in Posit Cloud, you‚Äôll land in a tool called RStudio.

RStudio is an IDE‚Äîshort for Integrated Development Environment. That just means it‚Äôs a workspace designed to help you write, run, and manage code, all in one place.

When you first open RStudio, you‚Äôll see three panels:

  1. Console and Terminal (left side). This is where R runs your code and shows the output‚Äîresults, messages, and errors. You might also see a Terminal tab‚Äîfeel free to ignore that for now.

  2. Environment / History (upper right). This panel shows what R currently ‚Äúknows.‚Äù

    • Environment: a list of all the objects (like data sets or variables) in your current session
    • History: a record of the commands you‚Äôve run
    • You won‚Äôt need to manage these tabs actively, but they‚Äôre useful for checking what‚Äôs going on.

  3. Files / Plots / Packages / Help / Viewer (lower right). This panel contains several tabs:

    • Files shows the files in your project
    • Plots displays any charts you generate
    • Packages helps you manage extra tools for R
    • Help is your built-in reference guide
    • Viewer lets you preview interactive content later on

That‚Äôs a lot of tabs and panes. We‚Äôll introduce each tab as we need it ‚Äî no need to memorize them all now.

Understanding Paths (Even If You Never Type One)

As you progress this course, you‚Äôll work with data files, save your results, and generate visualisations. All of that happens inside a project‚Äîa self-contained folder that holds everything you need.

When you open a project in Posit Cloud, something helpful happens behind the scenes: Posit Cloud automatically sets your working directory to the root of your project. That‚Äôs a fancy way of saying: ‚ÄúWhen R looks for a file (like a dataset), it starts looking inside this project and its sub-folders.‚Äù A project, in this case, just means a folder that contains everything related to a piece of work‚Äîyour data, code, results, and notes. Think of it like a digital binder that keeps all the pieces of an assignment in one place. When you open a project, R knows to look for files inside that folder‚Äîand nowhere else.

The Headache You Never Knew You Avoided

If you‚Äôve never written code before, you might not realise this‚Äîbut finding and loading files is one of the biggest early frustrations in data analysis.

In most setups, you have to:

  • Figure out where your files are on your computer,
  • Type out long, exact folder paths,
  • And debug mysterious ‚Äúfile not found‚Äù errors when something goes wrong.

Posit Cloud takes care of all of that for you. By opening a project, you‚Äôve already told R where to look‚Äîand everything just works.

You won‚Äôt need to think about file paths much in this course. But you will be quietly doing things the right way.

Working with Files: Keep It Simple

üí° As a rule: Keep all your data files inside your project folder. That way, R always knows where to find them.

You can refer to files by name (like "sales_data.csv"), or by their path within the project if they‚Äôre in a subfolder (e.g., "data/sales_data.csv").

No need to write long file paths like C:/Users/YourName/Desktop/... ‚Äî those won‚Äôt work for others and break the idea of reproducibility.

Stick to filenames and project-relative paths, and everything will stay clean, portable, and easy to run on any computer.

3.3 Our First Steps with R

We‚Äôve learned where your projects live, how Posit Cloud helps manage your files, and how RStudio is set up. Now it‚Äôs time to try writing some actual R code.

So what is R?

R is a programming language built for working with data. You can use it to calculate values, summarise information, create visuals, or build statistical models. And just like a calculator, it can respond immediately when you ask it to do something.

Let‚Äôs start by using R like a calculator. In the Console, you can type code and press Enter to run it. You‚Äôll see the result appear immediately below.

Try typing this in the Console:

2 + 2

And after pressing Enter, you should see the following returned to screen:

[1] 4

The [1] just means ‚Äúthis is the first item in the result.‚Äù You can ignore it for now.

So far, we‚Äôve just typed code and seen the answer appear in the Console. That‚Äôs useful for quick calculation‚Äîbut what if you want to use that result again later?

In R, you can assign a value to a variable using the arrow operator <-. For example:

result <- 2 + 2

This stores the result of 2 + 2 in a variable called result.

Now if you type:

result

you‚Äôll see:

[1] 4

Why is this useful?

  • You don‚Äôt have to repeat calculations
  • You can build more complex code step by step
  • It makes your work easier to read and modify
Variables as Labelled containers

Think of a variable as a labelled container: you give something a name so you can refer to it later.

Shortcut Key: The Assignment Operator

You can type the assignment arrow <- manually, but there‚Äôs also a handy shortcut:

  • Windows: Alt + -
  • Mac: Option + -
  • Linux: Alt + -.

Try using it as you type to save time!

From Vectors to Datasets

So far, we‚Äôve worked with a single vector‚Äîjust the ladder points. But most real-world data comes in tables, where each row represents an observation (like a team), and each column represents a variable (like their points).

Let‚Äôs turn our two vectors‚Äîteam names and ladder points‚Äîinto a mini dataset.

To turn this into a proper table, we‚Äôll use a tool from the tidyverse‚Äîa collection of R packages that make data analysis more consistent and readable. We‚Äôll need to first load the tidyverse package and then use a function provided by the tidyverse called tibble which combines R objects into datasets.

library(tidyverse)
ladder <- 
    tibble(team = team_names, 
           points = ladder_points
           )

ladder
# A tibble: 4 √ó 2
  team                points
  <chr>                <dbl>
1 Collingwood Magpies     60
2 Adelaide Crows          46
3 Brisbane Lions          58
4 Western Bulldogs        48

What is a tibble? That seems like a fair question to as. In short, A tibble is a modern, tidy version of a data frame. It looks like a spreadsheet but works in a more predictable way. You will use tibbles throughout this course.

What‚Äôs a Package?

In R, a package is a bundle of code, functions, and data designed to make your life easier. Instead of writing everything from scratch, you can load a package and get access to pre-built tools.

To use a package, you first install it (which we‚Äôve done for you), and then load it with library(package_name). After that, you can start using the functions it provides.

What is the tidyverse?

The tidyverse is a collection of R packages designed to work together seamlessly for data heavy tasks. These packages share a common philosophy:

  • Use clear, readable code
  • Handle data in a consistent and predictable way
  • Encourage best practices for reproducibility

You don‚Äôt need to learn all of the functions now ‚Äî we‚Äôll introduce them bit by bit as the course progresses.

From Console to Script: Why Save Your Code?

So far, you‚Äôve been writing code directly into the Console. That‚Äôs a great way to experiment and test things quickly. But there‚Äôs one big problem: When you close your session, everything you typed into the Console disappears.

That means:

  1. Your code isn‚Äôt saved
  2. You can‚Äôt easily reuse or edit it later
  3. It‚Äôs harder to share or get help

That‚Äôs why the next step is to start using R scripts ‚Äî files where you can write and save your code. Scripts help you:

  1. Keep a clean, editable history of your work
  2. Break your work into reusable steps
  3. Make your work reproducible for others (and your future self!)

Now that we know why it‚Äôs better to save your code, let‚Äôs learn how to do it.

How to Open a New R Script

  • In RStudio, go to the File menu
  • Choose New File > R Script
  • A blank window will open in the top-left panel ‚Äî this is your script editor

You can now type your code here instead of the Console.

Let‚Äôs copy some of the code we used earlier into the script:

library(tidyverse)

team_names <- c("Collingwood Magpies", "Adelaide Crows", 
                    "Brisbane Lions", "Western Bulldogs")
ladder_points <- c(60, 46, 58, 48)

ladder <- 
    tibble(team = team_names, 
           points = ladder_points
           )

ladder
# A tibble: 4 √ó 2
  team                points
  <chr>                <dbl>
1 Collingwood Magpies     60
2 Adelaide Crows          46
3 Brisbane Lions          58
4 Western Bulldogs        48

To run a line (or selection of lines), you can:

  • Highlight it and click Run in the top right of the script panel, or
  • Use the shortcut: Ctrl + Enter (Windows/Linux) or Cmd + Enter (Mac)

This sends the code to the Console and runs it as before ‚Äî but now your code is saved.

Don‚Äôt forget to save your script, with a name like ladder_analysis.R using File > Save As. You can reopen it anytime to continue working.

Getting Help in R

R has built-in documentation for almost every function. If you want to learn how a function works (or what arguments it takes), you can use the ? symbol followed by the function name. For example, suppose we want to find the minimum number of points in the dataset we have created. If we know the function to do this is min() but we dont know how to use it we can type the following into the Console:

?min

This opens the help page for the min() function, which finds the minimum value in a vector. We can use it as follows:

min(ladder_points)
[1] 46

on the vector ladder_points. If we needed to use it on the dataset ladder, then we need to access the column of points in the dataset. You can do that as follows:

min(ladder$points)
[1] 46

where the $ operator lets you extract a specific column from a dataset.

What if I don‚Äôt know the function name?

Sometimes you know what you want to do, but you don‚Äôt know the name of the function.

Here are a few ways to figure it out:

  • Use the Help panel (lower right in RStudio) and type keywords into the search bar.
  • Try the ?? operator to search function names and help pages in the Console. For example:
??minimum

This shows help files related to the word ‚Äúminimum.‚Äù

  • Use Google or search ‚ÄúR how to [do the thing]‚Äù

Over time, you‚Äôll learn common function names. But it‚Äôs totally normal to look them up ‚Äî everyone does, even the experts.

3.4 From Script to Story: Writing with Quarto

You‚Äôve seen how scripts help you save and structure your code. But what if you want to share your analysis‚Äîcomplete with notes, explanations, and results‚Äîall in one place?

That‚Äôs where Quarto comes in.

Quarto lets you combine code, output, and written explanation in a single document. It‚Äôs like a notebook that tells the full story‚Äînot just what you did, but why it matters.

A Quarto document is a special kind of file (with a .qmd extension) that:

  • Lets you write regular text using Markdown
  • Lets you include code chunks that run in R
  • Shows both the code and the results/output

We‚Äôll use Quarto documents to write up your work, show your results, and explain your reasoning.

Why would we want to use Quarto. There‚Äôs at least 3 good reasons why we might prefer Quarto to scripts:

  1. Documentation + Analysis: Keep your code and your thinking in one place.
  2. Reproducibility: Anyone can re-run your analysis and get the same results.
  3. Communication: Great for assignments, reports, and portfolios.

Creating Your First Quarto Document

Creating a Quarto document in RStudio is simple. Here‚Äôs how to get started:

  • In the RStudio menu, go to File > New File > Quarto Document‚Ä¶

A dialog box will appear. You can:

  • Give your file a Title (e.g. ‚ÄúAFL Ladder Summary‚Äù)
  • Leave the Author field blank or enter your name
  • Leave Format as HTML for now
  • Deselect the ‚ÄúUse visual markdown editor‚Äù
  • Click Create

You‚Äôll see a new file appear in the top-left pane with some example text included.
This is your Quarto source file, and it will have a .qmd extension (short for Quarto Markdown).

This is where you‚Äôll write your analysis: both the R code and the explanation that goes with it.

We‚Äôll now walk through the key elements of a Quarto document.

Top Matter

At the very top of the Quarto document, we can see a small block that looks like this:

---
title: "AFL Ladder Summary"
format: html
---

This section is called the YAML front matter (or just ‚Äútop matter‚Äù). It tells Quarto how to render your document ‚Äî like the title, output format, and other settings.

It always starts and ends with three dashes (---), and everything inside follows a key: value format.

We can add some key-value pairs to the top matter. Let‚Äôs add the date as follows:

---
title: "AFL Ladder Summary"
date: today
format: html
---

Now that we have some content inside our Quarto document, we want to view the output. Click the Render button at the top of the script panel. Quarto will run the code, insert the results into your document, and generate a clean HTML page. In Posit Cloud, the rendered HTML will open in a new browser tab. You can scroll through it to check that everything looks as expected‚Äîtext, code, and results all in one place. If something doesn‚Äôt look right, just go back to your .qmd file, make changes, and click Render again. It‚Äôs like refreshing your work after an update.

Essential Syntax for Quarto

When writing in Quarto, you‚Äôll use a format known as Markdown to format your text. Here are the essentials you‚Äôll need throughout this course:

Emphasis in text

Use * or _ for emphasis:

  • Bold:
    **This is bold** ‚Üí This is bold

  • Italics:
    *This is italic* ‚Üí This is italic

You can even combine them:
***Bold and italic*** ‚Üí Bold and italic

Headers

Headers help structure your document. Use # symbols:

# This is a level 1 heading
## This is a level 2 heading
### This is a level 3 heading

Note that a level 1 heading is a title in Quarto. So you should only use level 2 headings or below.

Ordered and Unordered Lists

Unordered List (with - or *)
- Apples
- Bananas
- Oranges

Produces:

  • Apples
  • Bananas
  • Oranges
Ordered List (with numbers)
1. First item  
2. Second item  
3. Third item

Leads to:

  1. First item
  2. Second item
  3. Third item
Nested (Sublists)
- Fruits
  - Apples
  - Oranges
- Vegetables
  - Carrots
  - Spinach

Yields:

  • Fruits
    • Apples
    • Oranges
  • Vegetables
    • Carrots
    • Spinach

Use two or four spaces to indent sub-items.

Links (URLs)

To include a link, use square brackets for the text and parentheses for the URL:

[Visit the AFL website](https://www.afl.com.au)

Produces:

Visit the AFL website

Paragraphs

Just leave a blank line between paragraphs:

This is the first paragraph.

This is the second paragraph.

Quarto will automatically format this with proper spacing in your final document:

This is the first paragraph.

This is the second paragraph.

Working with R Code Chunks

In a Quarto document, code lives inside chunks. These are special blocks that run R code and display the results right alongside your writing.

Here‚Äôs what a basic chunk looks like:

```{r}
mean(c(1, 2, 3, 4, 5))
```

This chunk will:

  • Run the code
  • Show the code
  • Show the result (which is 3)

i.e. we get the following:

mean(c(1, 2, 3, 4, 5))
[1] 3

By default, both the code and the result will appear in the final document.

Customising Code Chunks

Sometimes, you don‚Äôt want everything to be shown. Maybe you want to hide the code, or skip running it entirely. That‚Äôs where chunk options come in.

Chunk options start with #| and go inside the chunk, like this:

```{r}
#| eval: false

mean(c(1, 2, 3, 4, 5))
```

When we include that chunk in our document:

mean(c(1, 2, 3, 4, 5))

Notice how now the code was shown, but there was no output printed afterwards? That‚Äôs because the #| eval: false instructs Quarto not to run the code inside this chunk.

echo: Show or hide the code

We can do the opposite to displaying code and no output. Using:

#| echo: false

leads to:

  • The code will run
  • The result will appear
  • But the code won‚Äôt be shown

This is useful when you want the results to appear, but don‚Äôt want to show the underlying code.

warning: Suppress warning messages

When we add:

#| warning: false
  • The code will run normally
  • Any warnings will be hidden in the output

This keeps your output clean when R gives non-critical warnings that you want to suppress. We haven‚Äôt come across any of these yet, but when we do we will come back to this option.

We might stumble across other chunk options as well as we progress through the class:

  • message: false ‚Äî hides startup or package-loading messages
  • include: false ‚Äî runs the code, but hides both the code and the output (great for setup chunks)

You don‚Äôt need to memorize all of these now ‚Äî we‚Äôll remind you when they come up.

Writing Equations in Quarto

In Quarto, you can write mathematical equations using the same syntax used in LaTeX. There are two main ways to do this.

Inline Math

Use single dollar signs $...$ to include math inside a sentence:

The average is given by $\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$.

This renders as:

The average is given by \(\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i\).

Display Math

Use double dollar signs $$...$$ to write standalone equations on their own line:

$$
\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i
$$

This will render as:

\[ \bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i \]

Math is kept to a minimum in this book. As a result you won‚Äôt need to learn the too much of these details. Think of this as a reference in case at some point in your study you want to write equation in a Quarto document.

8. Search for what you‚Äôre trying to do‚Äînot just the error
